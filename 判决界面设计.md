⚖️ 《I Wanna Be A Judge》游戏化界面设计方案
1. 核心设计理念：从“审阅”到“审理”
将玩家的身份从“裁判文书读者”转化为“主审法官”。界面不再是线性的网页，而是一个铺满证据、档案和法律条文的法官办公桌面。
2. 核心界面模块设计
A. 被告人“犯人档案” (Criminal Profile Card)
将原本一段式的基本信息拆解为游戏角色词条。
视觉表现：一张带有红色“CONFIDENTIAL”印章的纸质档案。
游戏化词条 (Tags)：
基础属性：姓名（自报）、性别、年龄、职业（如：个体、无业）。
被动天赋 (Legal Status)：
🔵 初犯 (无前科) -> 量刑修正：0.9x
🔴 累犯 (五年前科) -> 量刑修正：1.2x
🟣 特殊体质 (如：聋哑人) -> 量刑修正：0.7x (法律强制减轻)
历史战绩：展示前科次数，如“盗窃罪 x 3”。
B. 证据证言“线索墙” (Evidence Dossier)
将 经审理查明 和 证据列举 转化为可点击或滑动的线索。
视觉表现：桌面上的多张照片快照、物证袋或证人笔录。
游戏化词条：
涉案金额：如“¥5839 (数额较大)”。
作案手段：如“扒窃”、“暴力抗法”、“酒后”。
损害评估：如“轻微伤”、“交通肇事逃逸”。
C. 法律天平“博弈区” (Judgment Balance)
这是核心的操作区，玩家通过勾选发现的“因子”来平衡法律的天平。
左托盘 (加重因子/红色)：
[ ] 主犯 [ ] 累犯 [ ] 拒不认罪 [ ] 社会影响恶劣
右托盘 (减轻因子/绿色)：
[ ] 自首 [ ] 立功 [ ] 退赃 [ ] 未遂 [ ] 如实供述

D. 财产刑“罚金刻度尺” (Financial Penalty Meter)
把“罚金/附加刑”从判决里单独拎出来做成一个更有操作感的模块：玩家不仅判“关多久”，还要判“罚多少”。
- 视觉表现：桌面上的“罚金刻度尺/计数器”，旁边放一叠钞票/罚单夹，刻度会随着量刑因子变动。
- 交互方式（建议二选一或组合）：
  - 罚金滑条：区间可按案件类型动态变化（如 0～50000 / 0～200000），并显示常见档位按钮（¥1000 / ¥3000 / ¥10000 / ¥20000）。
  - 罚金推荐档：系统根据已勾选因子给出“建议档位”，玩家可接受/上调/下调。
- 游戏化反馈：
  - 若案件存在“退赃/退赔/积极赔偿”，刻度尺会出现“绿色阻尼”，提示罚金通常会更低/更易缓刑。
  - 若案件存在“累犯/多次/社会影响恶劣”，刻度尺出现“红色超载区”，提示罚金更可能偏高。
3. 游戏化流程设计
第一阶段：阅卷 (Review Phase)
玩家点击桌面上的不同档案：
看身份：确认被告人是否具有特殊减免身份。
看事实：在长文本中点击高亮关键词，收集“量刑因子”。
（可增强游戏性）将“量刑因子”做成可收集的卡牌：玩家在案卷里点亮关键词后，获得一张卡牌（如：累犯/未遂/退赃/聋哑人），卡牌可拖拽到天平两侧。
第二阶段：裁决 (Judgment Phase)
玩家在“判决书”上进行填空操作：
罪名选择：从常见罪名池中拖拽罪名（如：盗窃罪、危险驾驶罪）。
刑期拨盘：通过一个旋转的物理拨盘或滑动条选择刑期，天平会随着玩家勾选的“因子”实时晃动。
罚金刻度尺：通过滑条/档位按钮选择罚金数额（若“附加刑/罚金”为无，则默认锁定为“无罚金”，但允许玩家误判以形成反馈）。
第三阶段：宣判与反馈 (The Verdict)
法槌视觉：提交时触发法槌敲响的音效和视觉震动。
评分系统 (Judge Rating)：
准确性 (Accuracy)：罪名是否完全一致。
平衡度 (Balance)：刑期与实际判决的偏差值（偏差在10%以内判定为“金牌法官”）。
财产刑命中 (Fine Accuracy)：罚金是否命中（可按“是否有罚金/金额区间/金额误差”三档评分）。
综合判决分 (Final Score)：可用加权方式，例如：罪名 50% + 刑期 35% + 罚金 15%。
对比揭秘：展示现实中法官的“认定理由”，作为攻略解析。
（可增强游戏性）结算页增加“法官称号/连胜机制”：
- 连续 3 次“罪名准确 + 刑期误差≤10%”授予称号（如：实务派法官）。
- 连续 3 次“罚金命中 + 量刑因子勾选正确率高”授予称号（如：财产刑专家）。
4. 需要支撑的数据词条（需进一步拆分）
为了实现上述设计，建议在现有的 CSV 数据基础上，进一步拆分或补充以下结构化词条：
词条名称	游戏表现	现状	补充建议
is_recidivist	累犯标识 (红色标签)	混合在 SECTION_1	需正则提取“系累犯”字样。

5. 沉浸感氛围建议 (Atmosphere)
色调：深木色桌面，沉稳的深蓝或灰黑色背景。
交互音效：翻页声、敲击键盘声、远处微弱的法庭背景杂音、标志性的法槌声。
文字特效：法官的认定理由可以像“卷轴”一样慢慢展开，而不是瞬间弹出。
（可增强游戏性）把“判决主文”做成盖章流程：先生成判决书草稿 → 玩家点击“盖章” → 显示真实判决主文（并高亮差异），强化仪式感。
